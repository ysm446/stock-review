# AGENTS.md

## 作業開始時の確認

作業前に、まず以下を確認する。

1. `docs/plan/goals.md`
   - プロジェクトの目的、完成形、重視する価値を把握する。

2. `docs/plan/plan.md`
   - 実装方針、優先順位、今後の予定を把握する。

3. `docs/plan/progress.md`
   - 現在の進捗、完了済み作業、未完了作業、注意点を把握する。

今回の依頼が現在の計画や進捗のどこに関係するかを把握してから作業する。方針と矛盾しそうな場合は、実装前に確認する。

## 変更を加えたら

コードや仕様に変更を加えたら、以下を更新する。

1. `docs/changelog.md`
   - ユーザー視点の変更点を記録する。日付見出し（`## YYYY-MM-DD`）の下に「追加 / 変更 / 修正」などで簡潔にまとめる。同じ日付の見出しが既にあれば、そこに追記する。

2. `docs/plan/progress.md`
   - 進捗・完了/未完了・注意点を最新の状態に保つ。

## このリポジトリでの進め方

- UI 文言・コメントは日本語を基本とする。
- フロントエンドは `src/`、Electron メインプロセスは `electron/`、Python バックエンドは `backend/` を中心に確認する。
- Python はプロジェクト直下の `.venv` を前提にする。
- ローカルデータは `data/` 配下に保存されるため、個人データを不用意に変更・削除しない。
- 変更後は、可能な範囲で該当画面やコマンドを確認し、確認できなかったことは報告する。
