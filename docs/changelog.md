# Changelog

## 2026-07-05

### 変更（UI 整理: ダッシュボード化）
- **ポートフォリオをダッシュボード化**: 1920x1080 のコンテンツ領域に全体が収まるレイアウトに変更。評価額推移・保有割合・銘柄別損益の3チャートを1行に小さく並べ、保有銘柄とウォッチリストを並列配置（テーブルは内部スクロール）。1500px 未満の画面では従来どおりの縦積み＋ページスクロールに自動フォールバック。
- **個別銘柄レビューを2カラム化**: 左に指標グリッド（概要・バリュエーション・収益性・財務サマリー・アナリスト情報）、右に銘柄チャットを全高で配置。チャットの入力欄が常に画面内に収まる。
- **最新ニュースパネルを削除**（レビュービュー）。ニュースはチャットの「ニュース検索」ツールに役割を移行。
- **銘柄チャットの会話を削除できるように**: 会話一覧の各項目に削除ボタン（×、確認ダイアログ付き）を追加。
- **銘柄チャットの回答を Markdown 表示に**（全体チャットと同じレンダラーを共通化、`src/chat-markdown.js`）。
- ビュー切り替え時にチャートを再描画し、非表示中に描画されたキャンバスが潰れる問題を修正。
- 開発用: `STOCK_REVIEW_AUTOSHOT=1` で起動すると各ビューのスクリーンショットを撮って自動終了（レイアウト検証用）。

### 追加（チャット再設計・第2弾: 役割ベースモデル管理）
- **モデルを2役割で運用**: 「常駐」（:8091、軽量モデル。ノート要約などの背景処理と、深堀り停止時のチャットのフォールバック）と「深堀り」（:8092、高性能モデル。チャット優先、VRAM 節約のため手動でロード/アンロード）。チャットは深堀り優先で自動フォールバックし、使用モデル名を回答メタに表示。
- **モデル設定モーダルを役割別 UI に刷新**: 役割ごとにモデル（GGUF）とコンテキスト長を選択し、起動/停止をワンクリックで。設定は保存され、常駐モデルはアプリ起動時に自動起動。
- **銘柄ノート要約が高速化**: 要約は常駐モデル + 思考無効（`enable_thinking:false`）で実行（実測 0.5 秒。思考有効だと約1000トークンの思考が先行する）。
- 旧・単一サーバー構成（:8080）からは自動移行（選択中モデルは「深堀り」に引き継ぎ）。
- ポート割り当てを news-picker と衝突しないよう調整（news-picker が 8081/8082 を使用中のため 8091/8092 を採用）。

### 追加（チャット再設計・第1弾）
- **エージェンティックチャット**: チャット（全体・銘柄別とも）が Web 検索・ニュース検索・銘柄指標取得をツールとして自律的に使い、出典 URL 付きで回答するようになった（`POST /chat/agent-stream`）。最大8ターンのツール使用、失敗時はモデルに伝えて続行、上限到達時は手持ち情報で回答を強制。
- **ツール活動と思考の表示**: 回答中に「🔍 ニュース検索: …→ N件」の活動ログと、折りたたみ式の「思考」（reasoning モデルの思考過程）を表示。
- **Web 検索**: `ddgs`（DuckDuckGo、API キー不要）による一般検索とニュース検索（`backend/search_web.py`）。検索失敗・0件は静かに空を返し、会話は止めない。
- **Ornith-1.0 対応**: llama-server を `--jinja` で起動し、reasoning（`reasoning_content`）とツールコールのテンプレート解析を有効化。サンプリング既定 {temp 0.6, top_p 0.95, top_k 20}、チャットは max_tokens 4096/ターン。Ornith 9B/35B の GGUF を `models/` に配置済みで、モデル選択モーダルからロード可能。

### 修正
- 会話メモリ・文書の全文検索（FTS5）に `ORDER BY rank` が無く、LLM に渡すコンテキストの選定が実質ランダムだった問題を修正。
- `GET /documents/search` が `/documents/{id}` に飲まれて常に 422 を返していた問題を修正（ルート登録順）。
- 銘柄別チャットで最終 SSE イベントを取りこぼすことがある問題を修正（末尾バッファのフラッシュ）。
- 未使用の `backend/fetch_prices.py` を削除。

### 追加
- **F12 スクリーンショット**: F12 キーでコンテンツ領域のスクリーンショットを `data/screenshots/YYYYMMDD-HHmmss.png` に保存。
- ウィンドウのコンテンツ領域を 1920x1080 基準に変更（`useContentSize`。スクリーンショットも同サイズになる）。
- `portfolio.json` などのデータファイルを保存時に1世代バックアップ（`.bak`）。破損時は自動的にバックアップから復旧。

### 修正
- **同一銘柄の複数ロットがローカル DB 上で1件に潰れる問題を修正**。`holdings` テーブルをロット単位のスキーマ（自動採番の主キー）へ移行し、DB 由来の読み込み・資産推移・配当集計がロットを正しく合算するように修正（既存 DB は起動時に自動移行）。
- **ロンドン市場など補助通貨単位（GBp=ペンス等）の銘柄が 100 倍で円換算される問題を修正**。GBp/GBX/ZAc/ILA を主要通貨に正規化してから換算。
- 評価額推移チャートが履歴不足時に架空のダミー系列（サイン波）を表示していた問題を修正。履歴が無い場合は「履歴がまだありません」と明示し、前日比・期間変動も表示しない。
- 通貨情報が取得できない場合のデフォルトが経路によって JPY / USD と不一致だった問題を修正。推測での換算をやめ、明示的にエラーとして銘柄ごとに表示。
- 過去の為替レートが欠損している日付に当日のスポットレートを使っていた問題を修正（直近過去のレートで補完）。
- 配当利回りが 1% 以下のとき 100 倍で表示されることがある問題を修正（桁推測ヒューリスティックを廃止）。
- 現金残高などの数値入力で全角数字が 0 円として保存される問題を修正（NFKC 正規化）。
- チャットの一部メッセージが文字化けして表示される問題を修正。
- 銘柄ノート（notes.md）や設定ファイルの書き込みをアトミック化し、クラッシュ時のファイル破損を防止。
- 保存処理の並行実行で「database is locked」により DB 同期が静かに失敗し得た問題を緩和（WAL + busy_timeout）。

### セキュリティ
- チャットサーバー（127.0.0.1:8001）に起動ごとに生成するトークン認証を追加し、CORS をアプリ（file:// 起点）のみに制限。ブラウザ上の任意サイトからチャット履歴・ノートの読み取りや削除・pip 実行系エンドポイントを叩けた問題を修正。
- ニュースタイトル・セクター名など外部（Yahoo/JPX）由来の文字列を HTML エスケープして描画（XSS 対策）。ニュースリンクは http(s) のみ許可。
- 新規ウィンドウ・外部ページへの遷移を禁止し、外部リンクは既定ブラウザで開くように変更（`setWindowOpenHandler` / `will-navigate` ガード）。
- Electron を 35（EOL）から 43 へ更新。
