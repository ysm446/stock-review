# Changelog

## 2026-06-29

### 変更
- 評価額推移チャートの「期間」（1ヶ月〜1年）と「軸」（相対軸 / 絶対軸）のプルダウン選択を localStorage に保存し、アプリ再起動後も維持するように変更。

### 修正
- 設定した現金残高（買付余力）がアプリ再起動後に 0 に戻ってしまう不具合を修正。現金は `portfolio.json` に `cash` キーで保存される一方、読み込み側が `cashJpy` キーだけを見ていたためで、両方のキーを受け付けるよう修正。
- 同一銘柄を複数ロット保有している環境で、ローカル DB への初回移行（レガシー JSON 取り込み）が重複ティッカーで失敗する不具合を修正（upsert 化）。

## 2026-06-28

### 追加
- ポートフォリオに現金残高（買付余力）を設定できるように追加。サマリーに「現金」と、株式評価額と合算した「保有資産評価（株式＋現金）」のカードを併記（従来の「総評価額」は「株式評価額」に名称変更）。
- 保有割合チャートに現金スライスを追加し、現金比率（キャッシュポジション）を可視化。
- 現金残高はローカル DB（`app_settings` テーブル）に保存し、再起動後も保持。

## 2026-06-26

### 追加
- 保有テーブル / 保有割合チャートの「同一銘柄をまとめる」と、保有割合チャートの配色モード（デフォルト / セクター別）を再起動後も保持するように変更。

## 2026-06-25

### 追加
- 設定ウィンドウを新設（左ナビ最下部の歯車アイコンから開く、タブ構成）。
  - **llama-cpp**: 最新リリースを確認し、Windows ビルド（CPU / CUDA / Vulkan ほか）をダウンロード・展開。CUDA は対応する cudart も同梱。
  - **埋め込み**: 埋め込みモデル（ruri-v3-310m）の状態表示と手動ダウンロード。依存（sentence-transformers ほか）が未導入なら、アプリ内から `.venv` へ自動インストール（PyTorch 込み）→ 続けてモデル取得。
  - **表示**: 「リソースモニターを表示」チェックボックス。ON で上部バーに CPU / RAM / GPU / VRAM の使用量をバー表示（依存 psutil / nvidia-ml-py は未導入なら自動インストール）。
- 保有割合チャートに配色プルダウンを追加（デフォルト配色 / セクター別グラデーション）。

### 変更
- 「保有銘柄数」を、重複保有を除いたユニーク銘柄数でカウントするように変更。
- llama-server のバイナリ保存先を `bin/llama-server/` から `runtime/llama-server/` へ移行（旧 `bin/` も後方互換で参照）。

### 修正
- 保有割合チャートで銘柄名ラベルが画面端で見切れる問題を修正。
- 保有割合チャートが広い画面で小さく見える問題を修正。キャンバス幅とドーナツ半径を表示領域に合わせて拡大するように変更。
- チャット画面で会話入力欄が見切れる問題を修正。会話（メッセージ）と会話ツリーだけが内部スクロールし、入力欄は常に表示。
- チャット画面の後続CSSで高さ指定が上書きされ、入力欄が再び見切れる問題を修正。

### その他
- `data/` 配下の個人データを git 管理対象外に変更（テンプレートと参照データは維持）。
- 開発計画ドキュメント（`docs/plan/`）と `CLAUDE.md` を整備。
- `CLAUDE.md` を参考に、Codex / エージェント向けの `AGENTS.md` を追加。
