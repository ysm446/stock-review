# Changelog

## 2026-07-13

### 追加
- **個別銘柄レビューにローソク足チャートを追加**: 始値・高値・安値・終値と出来高を表示。1か月 / 3か月 / 6か月 / 1年 / 全期間を切り替えられ、カーソル位置のOHLCVを確認できる。
- **株価履歴のローカル蓄積**: レビュー表示ごとに直近1年の日足を `app.db` の `review_price_history` へ追記。同じ日は最新値で更新し、取得範囲より古い日足は削除せず蓄積する。

### 変更
- **ローソク足の配色を赤・緑に変更**: 上昇日を赤、下落日を緑で表示するようにした。
- **ローソク足の縦軸を切りのよい目盛りに変更**: 価格帯に応じて `1 / 2 / 5 x 10の累乗` の間隔を自動選択し、軸の上下端も同じ間隔に揃えた。
- **山・谷の価格ラベルを追加**: 表示期間内の局所的な高値・安値を抽出し、重なりを避けて最大6点の価格をチャート上に表示。期間内の最高値・最安値は必ず含める。 山・谷の価格はどちらも明るいグレーで表示する。
- **移動平均線を追加**: MA25 / MA50 / MA75 を個別にオン・オフできるようにし、選択状態を localStorage に保存。表示期間より前の履歴も計算に使う。
- **個別銘柄レビューをキャッシュ先行表示に変更**: 前回取得した指標・財務・チャートを `app.db` から先に表示し、裏でYahoo Financeの最新データを取得して差し替えるようにした。更新中はチャートに表示し、通信失敗時も保存済みデータを残す。
- **起動時のPython依存関係確認を最適化**: `start.bat` でのpip実行を、`.venv` の新規作成時または `requirements.txt` の内容変更時だけにした。通常起動ではpipの確認ログを表示しない。
- **個別銘柄レビューの左カラムを上下分割**: 上部にローソク足チャートを固定し、下部の指標／ノート領域だけをスクロールする配置に変更。チャート上部に現在株価と前日比（金額・率）を表示する。

## 2026-07-12

### 変更
- **ノートへの反映を手動選択に変更**: 銘柄チャットの応答完了ごとに無条件でノートを自動更新する方式をやめ、各回答の下に「ノートに反映」ボタンを表示するようにした。反映したいやり取りだけを選んでノートに取り込める（過去の会話の回答にもボタンが付く）。反映中は「反映中...」、成功後は「反映済み」になる。うっかり関係ない話をしてもノートに混ざらない。
- **ノートを1世代バックアップ**: ノートの保存（反映・作り直し）前に、直前の内容を `notes.md.bak` へ退避するようにした。ノートタブの右端に「元に戻す」ボタンが出て、押すと保存前の内容に戻る（現在の内容とバックアップを入れ替える方式なので、もう一度押すと戻す前に戻れる）。
- **反映プロンプトに関連性フィルタを追加**: 銘柄の投資判断・企業分析に関係しない話題（雑談・操作の話など）はノートに書かず、反映すべき内容が無ければ既存ノートをそのまま出力するルールをマージプロンプトに追加。

### 追加
- **ノートの最終更新日時を常時表示**: 銘柄レビューのノートタブ右端に、`notes.md` の最終更新日時を表示するようにした。これまではその場で自動更新されたときだけ「HH:MM 更新」と出て、アプリを開き直すと消えていたが、ファイルの更新日時をバックエンドから返すことで銘柄を開いた時点から表示される。当日は「HH:MM 更新」、それ以前は「YYYY/M/D HH:MM 更新」形式。ノートがまだ空の銘柄では表示しない。

## 2026-07-10（第2弾）

### 変更
- **LLM モデル管理を単一サーバー構成にシンプル化**: 「常駐（standard）/ 深堀り（deep）」の役割ベース2ポート運用を廃止。起動直後は何もロードされておらず、上部バーの「モデルを設定」から `models/` フォルダの GGUF 一覧を表示 → クリックでロード → その1つのモデルがチャット・ノート更新などすべての処理を担当する。モーダルは「状態表示 + コンテキスト長選択 + 停止」と「モデル一覧（クリックでロード、稼働中表示）」に刷新。自動起動（常駐）設定は廃止。ポートは :8091 のみ使用（:8092 は解放）。旧設定からはモデル選択を自動移行し、アップグレード時に旧 standard サーバーが稼働中ならそのまま引き継ぐ。
- **ノートに出典URLを溜めない**: ノート自動更新・「ノートを作り直す」の両方に「出典URLの羅列（リンク集・出典セクション）を書かない」ルールを追加。会話を重ねるたびに出典リストが際限なく増えるのを防ぐ（重要な出典1〜2件を本文に添えるのは可。生の出典は会話ログに残る）。
- **テンプレート質問チップと入力欄の間隔を微調整**（少し空けた）。

### 修正
- **銘柄チャットが ROE 等の単位を誤読する問題を修正**: yfinance は ROE を小数（0.519 = 51.9%）で返すが、チャットのシステムプロンプトに生の値を渡していたため、LLM が「ROE 0.519%と極めて低い」のように誤解釈していた。PER/PBR/ROE/時価総額を画面表示と同じ整形済みの値（「51.9%」「76.07 倍」等）で渡すよう修正。あわせて「指標は Yahoo Finance 由来の参考値で、重要な判断では一次情報の確認を促す」注意書きをプロンプトに追加。

## 2026-07-07

### 変更
- **常駐 LLM（standard）を既定で自動起動しないように変更**: これまではバックエンド起動時に standard モデルを毎回自動起動（常駐）していたが、既定を OFF にした。モデル設定モーダルの standard カードに「バックエンド起動時に自動起動（常駐）」チェックボックスを追加し、ON にしたときだけ従来どおり常駐する。ノートPC 等で LLM を使わない/小さいモデルに切り替える運用に対応（設定は `llama_paths.json` のマシン固有設定として保存）。standard を要約・フォールバックに使う場合も、必要時に手動起動または要求時ロードで動く。

### 追加
- **データフォルダ（保存先ルート）の設定 + 初回セットアップ**（アプリとデータの分離）:
  - **初回起動フロー**: データ保存先が未設定のときは、メイン画面の代わりに初回セットアップ画面（`setup.html`）を表示。「フォルダを選択・作成…」から既存フォルダを選ぶか新規作成するまでメイン機能に進まない（**既定フォルダへの暗黙フォールバックは廃止** — 保存先を決めるまでアプリは開始しない）。設定後は自動でメイン画面へ遷移し backend を起動。
  - **保存先の変更**: 設定 > 「データ」タブで現在の保存先表示・変更（フォルダ選択/新規作成）・フォルダを開く。変更するとそのフォルダのデータに切り替わる（自動で再読み込み）。設定した保存先が見つからない（外付けドライブ未接続・削除など）ときも初回セットアップ画面に戻る。
  - パス解決を一本化: バックエンドは `backend/paths.py`、Electron は `electron/paths.js` に集約。Electron が環境変数（`STOCK_REVIEW_DATA_DIR` / `STOCK_REVIEW_CONFIG_DIR`）で Python 子プロセスへルートを渡す。
  - 環境設定の分離: `llama_paths.json`（マシン固有のモデルパス等）はユーザーデータルートに含めず、アプリ設定領域（`app.getPath("userData")`）へ分離（初回に旧 `data/llama_paths.json` から自動移行）。保存先ルートのポインタ（`config.json`）も同領域に置き、ユーザーデータの移動先に依存しない。
  - 注意: 変更は「保存先の切り替え」であり、既存データの自動コピー/移行は行わない（フォルダは事前に用意/コピーしておく）。指定フォルダが空なら初回に既定ファイルを作成する。
