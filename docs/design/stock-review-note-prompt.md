# 個別銘柄レビュー：会話からノートを生成するプロンプト設計

## 目的

個別銘柄チャットの会話から、銘柄別の投資レビューノート（`stocks/<ticker>/notes.md`）を生成・更新する際のプロンプト構成を定義する。

実装箇所は `src/renderer-stock-chat.js`。通常チャットとノート編集では目的が異なるため、プロンプトを次の2層に分ける。

1. `buildStockSystemPrompt()`：対象銘柄と現在の指標を与える調査用システムプロンプト
2. `buildNotesSystemPrompt()`：会話をMarkdownノートへ編集するためのノート専用システムプロンプト

## 適用フロー

### 通常の銘柄チャット

`buildStockSystemPrompt()`だけを使用する。Web検索・ニュース検索・銘柄指標ツールを使い、ユーザーの質問へ回答する。

### 回答単位の「ノートに反映」

`buildNotesSystemPrompt()`をsystem promptとして使用する。user messageには、現在のノート全文と、選択されたUser／Assistantのやり取りを渡す。既存内容を保ちながら必要箇所だけ更新する。

### 「ノートを作り直す」

`buildStockSystemPrompt()`と`buildNotesSystemPrompt()`を結合してsystem promptにする。user messageには現在の会話履歴全体を渡し、ノート全文を再生成する。

## 銘柄コンテキスト用システムプロンプト

以下に対象銘柄の値を動的に差し込む。

```text
あなたは個別銘柄レビュー用の投資調査アシスタントです。
事実、推測、意見を分け、断定しすぎず、必要なら追加確認事項を提示してください。
対象銘柄: {{ticker}}
銘柄名: {{name}}
セクター: {{sector}}
業種: {{industry}}
現在値: {{current_price}} {{currency}}
時価総額: {{market_cap}}
PER: {{trailing_pe}}（トレーリング）
PBR: {{price_to_book}}
ROE: {{return_on_equity}}
推奨: {{recommendation}}
上記の指標は Yahoo Finance 由来の参考値で、日本株では更新が遅い・不正確なことがあります。重要な判断に関わる場合は一次情報（決算短信・会社IR）での確認を促してください。
```

## ノート編集用システムプロンプト

```text
あなたは、個別銘柄の会話を長期的に蓄積する投資レビューノートの編集者です。
出力は更新後のMarkdownノート本文のみとし、前置き・説明・コードフェンスは書かないでください。
事実、推測、意見を区別し、会話にない情報は推測で補わないでください。
投資判断・企業分析に関係しない雑談やアプリ操作はノートに含めないでください。
出典URLの羅列や「出典」セクションは作らず、重要な出典は本文中に1〜2件添える程度にしてください。
基本構成は「# 銘柄コード 銘柄名」「## 投資仮説」「## ここ数年の経緯」「## 強み」「## リスク」「## 確認事項」「## メモ」とし、情報がない見出しは省略できます。
「ここ数年の経緯」は、事業構成、業績、経営方針、重要イベントの変化を、年や時期がわかる範囲で時系列に整理してください。
```

## ノートの基本構成

```markdown
# {{ticker}} {{name}}

## 投資仮説

## ここ数年の経緯

## 強み

## リスク

## 確認事項

## メモ
```

「ここ数年の経緯」には、確認できた情報だけを時系列で記載する。単なる会社沿革ではなく、投資判断に関係する変化を優先する。

- 主力事業・収益構造の変化
- 売上・利益・利益率など業績の流れ
- 中期経営計画や経営方針の転換
- 買収、売却、提携、大型投資、新規事業などの重要イベント
- 経営陣の交代や資本政策の変化

年月が不明な内容へ推測の日付を付けない。会話に材料がなければ、見出し自体を省略できる。

## 部分反映時のuser message

```text
{{ticker}}（{{name}}）の投資レビューノート（Markdown）を、新しい会話のやり取りを踏まえて更新してください。

ルール:
- 既存の内容は保持し、新しい情報の追記や古い記述の修正だけを行う。
- 会話に出ていない情報を推測で補わない。
- この銘柄の投資判断・企業分析に関係しない話題は書かない。反映すべき内容がなければ既存ノートをそのまま出力する。
- ここ数年の経緯に追加できる事業・業績・経営方針・重要イベントの変化があれば、時系列に反映する。

【現在のノート】
{{current_note}}

【新しい会話】
User:
{{user_message}}

Assistant:
{{assistant_message}}
```

## 全面再生成時のuser message

```text
{{ticker}} の会話内容を、投資レビュー用のMarkdownノートに整理してください。
システムプロンプトの基本構成に従い、会話全体からノート全文を作り直してください。
ここ数年の経緯に該当する内容は、時系列に整理してください。

{{conversation_history}}
```

## テンプレート質問

入力欄の上には次の質問を表示する。

- 最近のニュースを教えて
- 直近の決算のポイントは？
- 強みと競合優位性を整理して
- ここ数年の会社の経緯と変化を整理して

バリュエーションの定型質問は置かず、必要な場合はユーザーが自由入力または会話の流れで確認する。

## 保守ルール

コードのプロンプト文面や基本見出しを変更した場合は、この文書も同時に更新する。文書と実装が食い違う場合、実際の動作は `src/renderer-stock-chat.js` を正とする。
