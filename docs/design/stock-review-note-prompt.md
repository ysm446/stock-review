# 個別銘柄レビュー：会話からノートを生成するプロンプト設計

## 目的

個別銘柄チャットの会話から、銘柄別の投資レビューノートを生成・更新する際のプロンプト構成を定義する。

ノートは**カテゴリー別のカード**に分割して保存する（2026-07-19 変更。それ以前は単一の
`stocks/<ticker>/notes.md`）。実装箇所は `src/renderer-stock-chat.js`（プロンプト）、
`backend/chat_store.py`（カテゴリー定義と保存）、`backend/chat_server.py`（API）。

## カテゴリー（カード）定義

定義の実体は `chat_store.NOTE_CATEGORIES`（key / title / description）。
保存先は `stocks/<ticker>/notes/<key>.md`（カードごとに1世代の `.md.bak` バックアップ）。

| key | タイトル | 内容 |
| --- | --- | --- |
| business | 事業・製品 | 何で稼いでいる会社か、主力製品・サービス、ビジネスモデル、強み |
| financials | 業績・財務 | 決算の内容、売上・利益の傾向、財務体質、配当・株主還元 |
| recent | 最近の動き | 直近のニュース、発表、株価の材料になった出来事 |
| history | 経緯・歴史 | ここ数年の変化、転換点、過去の不祥事や構造改革 |
| environment | 競合・取引環境 | 競合他社、主要取引先、業界動向、規制やマクロの影響 |
| thesis | 投資判断メモ | 自分の考え。買い/売りの根拠、懸念点、監視ポイント |
| misc | その他 | 上のどれにも収まらない投資関連メモ |

分割前の旧 `notes.md` は当初「旧ノート（分割前）」として読み取り専用表示していたが、
ユーザー判断で同日中に**表示機能ごと廃止し、既存ファイルも削除済み**。
UI はニュースパネルと同様の小さめカードの行列（`repeat(auto-fill, minmax(250px, 1fr))`）で表示する。

## 適用フロー

### 通常の銘柄チャット

`buildStockSystemPrompt()`（銘柄コンテキスト）だけを使用する。Web検索・ニュース検索を使い、
ユーザーの質問へ回答する。

### 回答単位の「ノートに反映」（2段階）

1. **ルーティング**: 分類器システムプロンプト＋`buildRoutingPrompt()` で、会話が該当する
   カテゴリー key を1〜3個選ばせる（カンマ区切りのみ出力。反映すべき内容が無ければ `none`）。
   出力から既知 key を抽出し、1つも無ければ反映を中止してユーザーに通知する。
2. **カード別マージ**: 選ばれたカードごとに `buildCardEditorSystemPrompt()`＋
   `buildCardUpdatePrompt()`（そのカードの現内容＋新しい会話）でカード本文を再生成し、
   `PATCH /stocks/{ticker}/notes/{key}` で保存する。出力が既存内容と同一なら保存しない。

### 「ノートを作り直す」

`buildStockSystemPrompt()`＋再生成用システムプロンプトを結合し、会話履歴全体から
**カテゴリー見出し（`## 事業・製品` など、タイトルを一字も変えない）付きのノート全文**を
出力させる。出力を見出しで分割（`splitRegeneratedNote()`）し、カードごとに保存する。

- 未知の見出しのセクションは「その他」カードへ（見出しは太字行として残す）
- 最初の既知見出しより前の行（タイトル行など）は無視
- 出力に既知見出しが1つも無い場合は保存せずエラー表示
- 出力に現れなかったカードは変更しない

## 銘柄コンテキスト用システムプロンプト

以下に対象銘柄の値を動的に差し込む。

```text
あなたは個別銘柄レビュー用の投資調査アシスタントです。
事実、推測、意見を分け、断定しすぎず、必要なら追加確認事項を提示してください。
対象銘柄: {{ticker}}
銘柄名: {{name}}
セクター: {{sector}}
業種: {{industry}}
現在値: {{current_price}} {{currency}}
時価総額: {{market_cap}}
PER: {{trailing_pe}}（トレーリング）
PBR: {{price_to_book}}
ROE: {{return_on_equity}}
推奨: {{recommendation}}
上記の指標は Yahoo Finance 由来の参考値で、日本株では更新が遅い・不正確なことがあります。重要な判断に関わる場合は一次情報（決算短信・会社IR）での確認を促してください。
```

## 共通のノート編集ルール（NOTES_COMMON_RULES）

```text
事実、推測、意見を区別し、会話にない情報は推測で補わないでください。
投資判断・企業分析に関係しない雑談やアプリ操作はノートに含めないでください。
出典URLの羅列や「出典」セクションは作らず、重要な出典は本文中に1〜2件添える程度にしてください。
時期が分かる情報には年月を添えてください。
```

## カード編集用システムプロンプト（buildCardEditorSystemPrompt）

```text
あなたは、個別銘柄の会話を長期的に蓄積する投資レビューノートの編集者です。
ノートはカテゴリー別のカードに分かれており、あなたは指示された1枚のカードだけを編集します。
出力は更新後のカード本文（Markdown）のみとし、前置き・説明・コードフェンス・カードタイトルの見出しは書かないでください。
（+ 共通ルール）
```

## ルーティング用プロンプト（buildRoutingPrompt）

system: `あなたは投資ノートの分類器です。指示された形式だけで出力し、説明は書かないでください。`

```text
{{ticker}} についての以下の会話を、投資ノートのどのカテゴリーカードに反映すべきか選んでください。

カテゴリー一覧:
- business: 事業・製品 — …
（全カテゴリーを key: タイトル — 説明 の形式で列挙）

出力ルール:
- 該当するカテゴリーの key だけをカンマ区切りで1〜3個出力する（例: recent,financials）。
- 投資判断・企業分析に関わる内容が会話に無い場合だけ none と出力する。
- 説明や理由は一切書かない。

【会話】
{{exchanges}}
```

## カード更新時のuser message（buildCardUpdatePrompt）

```text
{{ticker}}（{{name}}）の投資ノートのうち「{{card_title}}」カード（{{card_description}}）を、新しい会話のやり取りを踏まえて更新してください。

ルール:
- このカードの範囲に該当する内容だけを反映する。他のカテゴリーに属する内容は書かない。
- 既存の内容は保持し、新しい情報の追記や古い記述の修正だけを行う。
- 会話に出ていない情報を推測で補わない。
- この銘柄の投資判断・企業分析に関係しない話題は書かない。反映すべき内容が無ければ既存の内容をそのまま出力する。
- 出典URLの羅列は書かない。特に重要な出典を本文中に1〜2件添える程度にとどめる。
- カードのタイトル見出しは書かず、箇条書き中心の簡潔なMarkdownにする。

【現在のカードの内容】
{{card_content}}

【新しい会話】
User:
{{user_message}}

Assistant:
{{assistant_message}}
```

## テンプレート質問

入力欄の上には次の質問を表示する。

- 最近のニュースを教えて
- 直近の決算のポイントは？
- 強みと競合優位性を整理して
- ここ数年の会社の経緯と変化を整理して

バリュエーションの定型質問は置かず、必要な場合はユーザーが自由入力または会話の流れで確認する。

## 保守ルール

コードのプロンプト文面やカテゴリー定義を変更した場合は、この文書も同時に更新する。
文書と実装が食い違う場合、実際の動作は `src/renderer-stock-chat.js` /
`backend/chat_store.py` を正とする。
