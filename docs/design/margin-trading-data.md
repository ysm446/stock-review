# 信用残データとチャート表示

株価チャートの「信用残」機能（2026-07-19 追加）で扱うデータの性質・入手事情・実装・読み方の注意をまとめる。

## 信用残とは

信用取引は、証券会社から資金や株式を借りて行う売買。その未決済の建玉残高が信用残で、
銘柄ごとに次の2つが公表される。

- **信用買い残**: 借りた資金で買った建玉の残高（株数）。将来の**売り圧力**（返済売り）になる。
- **信用売り残**: 借りた株を売った建玉の残高（株数）。将来の**買い圧力**（買い戻し）になる。
- **信用倍率** = 買い残 ÷ 売り残。1倍に近い・割るほど「取り組みが良い」と言われる。

制度信用（取引所ルール・6か月期限）と一般信用（証券会社ルール）の2種類があり、
公表値はその合計と内訳。本アプリは**合計のみ**を保存・表示している。

## データ入手の事情（2026-07 調査）

| 入手経路 | 粒度 | 過去分 | 費用 | 備考 |
| --- | --- | --- | --- | --- |
| JPX 銘柄別信用取引週末残高（PDF） | 週次・全銘柄 | **直近5週のみ** | 無料 | **採用**。削除済みの過去週はURL直打ちでも404 |
| Wayback Machine（上記PDFのアーカイブ） | 週次・全銘柄 | 約90週（2015〜16 / 2021〜22 / 2024〜25。クロール頻度依存で不連続） | 無料 | **バックフィルに採用**（`fetch_margin.py --backfill`）。旧フォーマットも同じパーサで解析可能なことを確認済み |
| J-Quants API（JPX公式） | 週次・銘柄別 | Standardで10年 / Premiumで20年 | Standard 月額約3,300円〜 | 連続した過去分を一括投入したくなったときの正攻法。Free/Lightプランでは信用残は取得不可 |
| 日証金 貸借取引残高 | 日次 | 銘柄検索で過去分あり | 無料 | 貸借取引（機関の貸借）ベースで信用残とは別物。CSRF付きセッションフォームで自動化が壊れやすく見送り |
| IRBank・かぶたん等の民間サイト | 週次 | 数年分あり | 無料 | JSによる動的読み込み＋利用規約の懸念で見送り |

- 公表スケジュール: 毎週金曜（申込日）時点の残高が、**翌週第2営業日（通常火曜）の夕方**に公表される。
- 無料の公式データでは過去へ遡れないため、**取得を始めた時点から蓄積していく**方式を採用した
  （日足の `review_price_history` と同じ発想）。過去5週分は初回取り込みで入る。

## 本アプリでの実装

### 取得と蓄積（backend/fetch_margin.py）

- JPXページから未取り込み週の PDF をダウンロードし、pdfplumber で解析して
  `app.db` の `margin_history`（`code` + `week_date` 主キー、売残・買残）へ**全銘柄分**を upsert する。
- PDF は「その週の全銘柄（約3,900銘柄）が1ファイル」なので、銘柄単位のダウンロードは存在しない。
  1回の取り込みで全銘柄が入るため、銘柄を切り替えても再ダウンロードは発生しない。
- 解析の検証則: 「売残合計 = 一般信用 + 制度信用（買残も同様）」の恒等式が成立しない行は
  列ずれの疑いがあるため破棄する（実測: 3,931行中3,929行一致・不一致0）。
- 銘柄コードは PDF の5桁表記から末尾の `0` を除いて正規化（`72030`→`7203`）。
  優先株など末尾が `0` でないコード（例 `25935`）はそのまま。ティッカー `7203.T` との照合に使う。
- JPX はデフォルトの python-requests UA を 403 で拒否するため、ブラウザ風 User-Agent を送る。

### 更新タイミング（専用ボタンなし）

次の既存フローに相乗りして自動実行される。ユーザー操作は不要。

1. 東証銘柄（`.T`）のレビューを開いたとき（背景のスナップショット更新のついで）
2. 「日足を再取得」ボタンを押したとき

`margin_meta.last_checked` による**6時間スロットル**付き。新データは週1回（火曜夕方）しか
出ないので、通常は週1本の PDF 取り込み（約10秒）だけが走る。初回のみ5週分で1分弱。
取り込み失敗はstderrに出すだけで、日足取得やレビュー表示は止めない。

### 過去分バックフィル（設定画面 または fetch_margin.py --backfill）

Internet Archive（Wayback Machine）がクロールした過去の週次PDFを一括で取り込む。

- **設定 > データ > 信用残の過去データ**: 期間（過去1年 / 2年 / 3年 / 全期間）を選んで
  「過去分を取り込む」を押す。進捗（n/m 週目）と完了サマリーがボタン下に表示される。
  実装は `POST /margin/backfill`（SSE、`{"since": "YYYY-MM-DD" | "YYYY" | null}`）→
  `fetch_margin.iter_backfill()`。
- CLI からも実行できる:

```
.venv\Scripts\python.exe backend\fetch_margin.py --backfill [--since 2024]
```

- CDX API でアーカイブ済みPDFの一覧を取得し、期間内で未取り込みの週だけをダウンロード・
  解析して `margin_history` へ upsert する。週ごとにコミットするので**中断しても再実行すれば
  続きから**進む（取り込み済みの週はスキップ）。
- アーカイブのクロール頻度に依存するため**連続した週次にはならない**
  （2026-07 時点: 2015〜2016年・2021〜2022年・2024年・2025年前半の約90週。
  2017〜2020年・2023年・2025年秋〜2026年春は欠落）。
- チャートの折れ線は**間隔が14日を超える週同士を結ばない**（欠測として分断。
  当初は古い値のフラット線で埋めていたが、誤解を招くため変更）。
  欠測区間はツールチップにも信用残を表示しない。
- archive.org への負荷を抑えるため週ごとに1秒の間隔を置く。1週あたり10秒前後。

### チャート表示（src/candlestick-chart.js）

- 「信用残」メニューで買い残（オレンジ `#f97316`）・売り残（ブルー `#38bdf8`）を個別にオン・オフ。
  選択は localStorage（`stock-review.reviewMarginSeries`）に保存。既定は両方オン。
- 出来高エリアに**週次の点（申込日＝週末の足の位置）を直線で結ぶ折れ線**で描画
  （当初は階段状だったが、傾向を追いやすい折れ線に変更）。各点に小さなドットを打ち、
  前後が欠測で結べない孤立した週もドットで見える。タイムマシン表示にも追従する。
- ツールチップに信用買残・売残・信用倍率（何週末時点かの日付付き）を表示。
  割り当ては「その日以前の最新週」（14日より古い値は欠測扱いで非表示）。
- データはレビュー/日足のペイロードに `marginHistory` として相乗りする
  （`fetch_review.py` / `review_cache.py`。IPC の追加なし）。マーケットページの指数・FXは
  対象外のため、メニュー自体を配置していない（config 省略で安全に無効になる）。

### スケールの扱い（重要）

出来高バーと信用残ラインは**描画エリアを共有するが、縦スケールは独立**。

- 出来高バー: 表示期間内の日次出来高の最大値基準
- 信用残ライン: 表示期間内の買い残・売り残（表示中の系列）の最大値基準

信用残は残高なので日次出来高より1桁前後大きいことが多く、スケールを共有すると
どちらかが潰れるため。帰結として **バーとラインの高さ比較に意味はない**。
ラインの形（増減・買い残と売り残の交差）と、買い残・売り残ライン同士の比較
（同一スケール）だけが意味を持つ。実数はツールチップで確認する。

## 読み方の注意

- **週次データ + 公表ラグ**: 最新値でも数日〜1週間古い。日中の需給は反映されない。
- **ほぼ「国内個人投資家」の需給を映す**: 信用残は日本の証券会社の信用取引口座の建玉。
  国籍での区別はないが、信用取引は本質的に個人向けの制度で利用者の大半は国内個人。
- **海外機関投資家の売買は含まれない**: ヘッジファンド等の空売りは貸株市場で株を借りて行う
  （または先物・オプション）ため、信用売り残には現れない。機関の買いも現物・先物中心。
  「売り残が少ないのに下がり続ける」場合、信用残の外側で機関の空売りが行われている可能性がある。
- 外国人・機関の動向を見たい場合の公的データ:
  - **空売り残高報告**（JPX、日次・銘柄別）: 発行済株式の0.5%以上の空売りポジションの開示。
    機関の空売りはこちらに出る。無料で過去分も取りやすい。
  - **投資部門別売買状況**（JPX、週次）: 「海外投資家が◯億円買い越し」の元データ。
    ただし市場全体の集計で銘柄別はない。

## 将来の拡張候補

- **空売り残高報告の蓄積・チャート重ね**: 機関の売り圧力を可視化。信用残（個人）と補完関係。
  銘柄別・無料・CSV で取得しやすく、`margin_history` と同じ蓄積方式が使える。
- **J-Quants Standard による連続した過去分の投入**: 契約すれば10年分（欠落なし）を
  `margin_history` へ一括投入できる。テーブル構造はそのまま流用可能。
  Wayback バックフィルの欠落期間（2017〜2020年・2023年等）を埋める唯一の現実的手段。
- 手動取り込みボタン（公表直後に今すぐ見たい場合）。通常は自動更新で足りる。
